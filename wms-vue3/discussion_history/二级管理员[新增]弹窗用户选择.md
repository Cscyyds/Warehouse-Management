# 二级管理员「新增」弹窗用户选择 — 方案讨论记录

## 需求描述

系统管理 → 权限管理 → 二级管理员，将"新增"功能从跳转页面改为弹窗形式，弹窗内做用户选择操作。
数据来源与角色管理关联，新增按组织/角色/用户类型三种维度的筛选。

---

## UI 参考图解析（image.png）

弹窗标题：**用户选择**，整体为三栏布局：

### 左侧面板（可收起）
- 顶部切换器：`按组织查询` / `按角色查询` / `按用户类型`
- 展示对应树形结构，点击节点过滤中间列表
- 有收起/展开面板的箭头按钮

### 中间面板（用户列表）
- 搜索栏字段：账号、昵称、姓名、机构（带搜索icon）、公司（带搜索icon）、手机 + 查询/重置
- 表格列：序号 / 登录账号 / 用户昵称 / 员工姓名 / 归属机构 / 归属公司 / 手机号码 / 更新时间 / 状态
- 分页（共24条，每页20条）

### 右侧面板（已选）
- 标题"当前已选择 N 项："
- 列出已选用户，可点 × 移除

### 底部按钮
确定 / 关闭

## 实现方案

### 改动范围

| 文件 | 改动内容 |
|------|---------|
| `src/views/system/Admin.vue` | `handleAdd` 改为打开弹窗；引入 AdminSelectDialog 组件 |
| `src/views/system/components/AdminSelectDialog.vue` | 新建，整个弹窗组件 |
| `src/api/modules/admin.ts` | 新增 `batchCreateAdmin` 接口（预留，暂不对接后端） |
| `src/api/modules/personnel.ts` | 确认 `getPersonnelList` 是否存在，不存在则补充 |

### 数据来源

| 筛选维度 | 数据来源 |
|---------|---------|
| 按组织查询（左侧树） | `getOrgTree()` |
| 按角色查询（左侧树） | `getRoleAll()` |
| 按用户类型（左侧树） | 静态选项：管理员 / 员工 / 访客 |
| 中间用户列表 | `getPersonnelList(params)` |

### 弹窗组件结构

```
AdminSelectDialog.vue
├── 左侧面板（可收起，宽约220px）
│   ├── 切换器：按组织 / 按角色 / 按用户类型
│   └── el-tree（根据切换器渲染不同树/列表）
├── 中间面板（flex: 1）
│   ├── 搜索栏（账号/昵称/姓名/机构/公司/手机）
│   ├── el-table（用户列表，点击行选中高亮）
│   └── el-pagination
└── 右侧面板（宽约180px）
    ├── 已选 N 项
    └── 已选用户列表（可点×移除）
底部：确定 / 关闭
```

### 交互逻辑

- 点击左侧树节点 → 自动带入对应筛选条件，刷新中间列表
- 点击表格行 → 加入右侧已选列表（再次点击取消选中）
- 点击"确定" → 批量提交已选用户为二级管理员
- "编辑"功能保持原有跳转页面方式不变

### 提交策略（已确认）

- **当前实现**：前端循环调用 `createAdmin`，逐条提交
- **预留接口**：`batchCreateAdmin`（`POST /system/admin/batch`），后端就绪后直接切换调用

```typescript
// admin.ts 预留接口
export function batchCreateAdmin(data: Partial<AdminItem>[]): Promise<ApiResponse<null>> {
  return post<null>('/system/admin/batch', data)
}
```

## 讨论结论

- 弹窗内展示的是**人员列表**（Personnel），不是角色列表
- 左侧"按角色查询"的角色选项从 `getRoleAll()` 获取
- 多选用户后批量添加为二级管理员
- 提交方式：前端循环调用，预留批量接口

### 3. `src/views/system/components/AdminSelectDialog.vue` — 新建弹窗组件

**组件职责：** 三栏布局弹窗，供选择人员后批量添加为二级管理员。

**关键数据流：**

| 数据 | 来源 | 用途 |
|------|------|------|
| `orgTree` | `getOrgTree()` | 左侧"按组织"树 |
| `roleList` | `getRoleAll()` | 左侧"按角色"列表 |
| `userTypeOptions` | 静态常量 | 左侧"用户类型"列表 |
| `tableData` | `getPersonnelList(params)` | 中间用户列表 |
| `selectedUsers` | 本地 ref | 右侧已选面板 |

**Props / Emits：**
```typescript
const visible = defineModel<boolean>({ default: false })  // v-model 控制显隐
const emit = defineEmits<{ (e: 'success'): void }>()      // 提交成功后通知父级刷新
```

**左侧三种筛选维度逻辑：**
- 按组织点击 → `filterForm.orgId` 带入查询，清空其他维度
- 按角色点击（再次点击取消）→ `filterForm.roleId` 带入查询，清空其他维度
- 用户类型点击（再次点击取消）→ `filterForm.userType` 带入查询，清空其他维度

**提交逻辑：**
```typescript
// 前端循环调用，后端批量接口就绪后替换为：
// batchCreateAdmin(selectedUsers.value.map(...))
const results = await Promise.allSettled(
  selectedUsers.value.map(user => createAdmin({ ...user, status: '正常' }))
)
```

**弹窗生命周期：** `watch(visible)` 监听打开事件，每次打开自动重置状态并重新拉取数据，`destroy-on-close` 保证关闭后 DOM 销毁。

## 变更记录 2 — 筛选方式重构 & 弹窗放大

### 变更内容

**UI 调整：**
- 弹窗宽度：`1100px` → `1300px`
- 弹窗内容区高度：`520px` → `640px`
- 表格高度：`360px` → `480px`
- 移除左侧可收起的 Tab 面板（按组织 / 按角色 / 用户类型）
- 改为在主区域顶部新增一行筛选下拉，布局变为两栏（主区域 + 右侧已选）

**筛选下拉行（新增）：**

| 控件 | 类型 | 数据来源 | 传参字段 |
|------|------|---------|---------|
| 按组织筛选 | `el-tree-select`（树形下拉） | `getOrgTree()` | `orgId` |
| 按岗位筛选 | `el-select` | `getPositionList({ page:1, pageSize:999 })` | `positionId` |
| 按角色筛选 | `el-select` | `getRoleAll()` | `roleId` |

三个下拉均支持 `clearable` + `filterable`，选中后立即触发 `handleSearch`，重置按钮一并清空。

**Script 变更：**
- 移除：`leftCollapsed`、`leftTab`、`userTypeOptions`、`handleOrgClick`、`handleRoleClick`、`handleUserTypeClick`
- 移除 icon 引入：`ArrowLeft`、`ArrowRight`
- 新增：`positionList`（`ref<PositionItem[]>`）、`fetchPositions()`
- 新增引入：`getPositionList`、`PositionItem`
- `filterForm` 字段：`{ orgId, roleId, userType }` → `{ orgId, positionId, roleId }`
- `init()` 并发拉取改为：`Promise.all([fetchOrgTree(), fetchRoles(), fetchPositions()])`
- `loadData` params 新增 `positionId: filterForm.positionId || undefined`
